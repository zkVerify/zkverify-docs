## Phase 2 - Build-to-Earn...more! + Galxe campaign
在技术向的 Phase 1 之后，Phase 2 正式开启！核心技术任务基本保留，同时向更广的 Web3 社区开放。

Phase 2 operates on two parallel tracks:

### Technical Track

- <b>Build a verification pallet:</b> Phase 2 继续开放，奖励提升至 50,000 分！ 
- <b>Integrate zkVerify:</b> 从零构建 ZK dApp 或将 zkVerify 接入现有项目用于验证，奖励随复杂度而定。 
- <b>Send proofs for verification:</b> 提交更多证明以展示集成，验证越多积分越高。 

|  Challenge | Points  | Status |
|---|---|---|
| Build a verification pallet  |  50'000 | Active  |
| Integrate zkVerify  | Depends on complexity  | Active  |

有问题？请在 zkVerify [Discord](https://discord.com/invite/zkverify) 联系我们！

### Community Track
不是 ZK 开发者？可 [参加 Galxe 活动](https://app.galxe.com/quest/QzaQxuvgdzSVXmvvcY5HHy/GCRCJtV8PN)，完成任务学习 ZK、分享知识、传播 zkVerify，完成任务可获积分。

### A single source of truth
我们构建了专用 dApp 汇总技术与社区贡献，无论是提交技术成果还是完成 Galxe 任务，进度都会同步，作为激励测试网的唯一“事实来源”。

可在综合排行榜查看排名、实时积分和他人对比。链接待公布。

## Phase 1 - Build-to-Earn
Phase 1 提供两项挑战：

- <b>Build a verification pallet:</b> 为基于 Substrate 的链开发高效验证 ZK 证明的关键 pallet。
- <b>Build a demo dApp:</b> 从零构建 demo dApp，集成 zkVerify 作为验证与结算层，展示实际用例；可以改造现有 dApp 使用 zkVerify 并部署，或升级为可在移动端快速生成的新型 zk-proof 并构建移动 dApp。

|  Challenge | Points  | Status |
|---|---|---|
| Build a verification pallet  |  5000 | Updated (Phase 2)  |
| Build a demo app  | Variable  | Updated (Phase 2)  |
